# Security Policy

## Reporting a vulnerability

Please use GitHub private vulnerability reporting for this repository when available.

Include:

- affected URL, file, or deployment;
- impact and realistic attack scenario;
- reproduction steps;
- minimal proof of concept;
- suggested remediation, when known.

Do not include secrets, private browsing records, unpublished manuscripts, personal data, or exploit details in a public issue.

## Scope

This repository contains the public static website. Browser extension and Zotero plugin vulnerabilities should be reported in their respective repositories.

## Response targets

We aim to acknowledge valid reports within seven days and provide a remediation plan or status update within fourteen days. These are targets rather than contractual service-level guarantees.
