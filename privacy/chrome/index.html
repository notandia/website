Privacy policy for MDPI Filter

Last updated: July 23, 2026

MDPI Filter identifies MDPI publications in search results, bibliographies, inline citations, cited-by lists, and similar-article lists. Most processing occurs locally in Chrome.

Information handled

The extension stores user preferences with the Chrome extension storage API. Chrome may synchronize those preferences when Chrome Sync is enabled. Content scripts inspect text, links, citation identifiers, and document structure on HTTPS pages you visit. Complete page content, browsing history, and reference text are not transmitted to the developer or to an analytics service.

The extension may extract DOI, PMID, and PMCID publication identifiers. Detection results and reference previews are held in bounded temporary memory caches and are not used for tracking.

External communications

When NCBI lookups are enabled, the extension sends only validated DOI, PMID, or PMCID values to the NCBI ID Converter endpoint over HTTPS. Requests identify the application as mdpi-filter, omit browser credentials and referrer information, and are deduplicated, batched, and rate-bounded. NCBI necessarily receives the identifiers and the IP address used for ordinary network communication.

NCBI lookups can be disabled in advanced settings. Disabling them results in no NCBI lookup requests but may reduce detection accuracy.

Selecting Report Filter Issue opens a public GitHub issue form. The extension pre-fills only the page origin and path, browser and extension version, and filter mode; it removes query parameters and fragments. Nothing is submitted automatically, and the user can edit or discard the report.

No analytics or commercial sharing

The extension contains no advertising or analytics service. The developer does not receive extension telemetry and does not sell, rent, or share page content, browsing history, publication identifiers, or preferences for advertising, profiling, or data-broker purposes.

Retention and control

  • Preferences remain in Chrome extension storage until changed, cleared, or the extension is removed.
  • Per-tab reference data is cleared when navigation begins or the tab closes.
  • The extension does not maintain a persistent browsing-history database.
  • Users can disable NCBI lookups, change site access through Chrome controls, clear extension data, disable the extension, or uninstall it.

Security and contact

The extension uses Manifest V3, a self-only content security policy, no remote executable code, validated cross-context messages, and no shipped runtime npm dependencies. Sensitive vulnerabilities should be reported through the private GitHub Security Advisory channel. Non-sensitive privacy questions may be submitted through the project’s public issue tracker.

No product analytics are used.
