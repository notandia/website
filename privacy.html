Privacy

Research checks should not become browsing surveillance.

This overview describes Notandia’s common privacy boundary. Each extension has a platform-specific policy whose terminology, storage behavior, and permission flow match that platform.

Notandia, previously distributed as MDPI Filter, is an independent open-source project. It is not affiliated with, authorized by, or endorsed by MDPI AG, Crossref, Retraction Watch, NCBI, browser vendors, or any publisher or data provider.

Local processing

The browser extension processes page structure, links, citation text, and scholarly identifiers locally to identify publisher-related content and prepare optional integrity checks. Settings are stored through the browser extension storage API. The project does not receive complete page text, browsing history, search queries, or extension telemetry.

NCBI detection lookups

When enabled, NCBI lookups transmit only validated DOI, PMID, or PMCID values required for publication resolution. Requests identify the application as Notandia, omit browser credentials and referrer information, and are deduplicated, batched, and rate-bounded. Disabling NCBI lookups causes zero related requests but may reduce MDPI-detection coverage.

Optional Crossref integrity lookups

This feature is off by default. After explicit user consent, the extension may transmit normalized DOI identifiers to Crossref solely to retrieve scholarly metadata and formal post-publication update relationships. It does not send the page address, article text, citation text, search query, browsing history, account identifier, or analytics identifier.

Each scan attempts no more than 50 unique DOI lookups and starts no more than four requests per second. Disabling the feature stops new requests and cancels active or queued work. Firefox presents an additional optional data-collection permission prompt; denial results in no Crossref requests.

Interpretation limits

The extension reports evidence found in checked records. A statement that no known signal was found is not a guarantee that a work is correct, reliable, or free of undiscovered concerns. Deferred, failed, and unresolved checks are disclosed rather than treated as clear.

Retention and user control

  • Preferences remain in browser extension storage until changed, cleared, or the extension is removed.
  • Per-tab reference and display data is cleared when navigation begins or a tab closes.
  • Integrity lookup responses may be cached only in extension memory for up to 24 hours and are not used to create a persistent browsing-history database.
  • Users can disable network-assisted features, change website access, deny optional permissions, clear extension data, disable the extension, or uninstall it.

No analytics or commercial sharing

The extension and website contain no advertising or product analytics service. The developer does not sell, rent, or share page content, browsing history, publication identifiers, or preferences for advertising, profiling, or data-broker purposes.

Website

The project website contains no analytics, advertising, account system, database, cookies, or remote JavaScript. Cloudflare may process ordinary network and security metadata needed to deliver the site under its infrastructure terms.

Reporting a privacy issue

Use the relevant repository’s private security process for sensitive reports. Do not post private browsing data, access tokens, unpublished manuscripts, or personal information in a public issue.
