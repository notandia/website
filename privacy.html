Privacy

Research checks should not become browsing surveillance.

This page describes the project’s privacy design direction and must be kept synchronized with the exact behavior of each released client.

Current MDPI detection

The browser extension processes page content locally to identify MDPI links and citations. Optional NCBI lookups may transmit scholarly identifiers required for resolution. Settings are stored through the browser extension storage API.

Planned integrity lookups

The planned work-level integrity layer transmits normalized scholarly identifiers such as DOI, PMID, or PMCID only when a lookup is enabled. It must not transmit complete page text, citation text, search queries, full browsing history, account identifiers, or analytics identifiers.

User control

  • Network-assisted lookups must have a documented control.
  • Disabling lookups must result in zero related external requests.
  • Coverage reductions caused by disabling lookups must be visible.
  • Private-browsing behavior must be platform-specific and opt-in where the browser requires it.

Website

The initial project website contains no analytics, advertising, account system, database, cookies, or remote JavaScript. Cloudflare may process ordinary network and security metadata needed to deliver the site under its infrastructure terms.

Platform disclosures

Chrome, Edge, Firefox, Safari, and Zotero documentation will be maintained separately when their runtime behavior differs. Store privacy forms must match observed network behavior before every release.

Reporting a privacy issue

Use the relevant repository’s security process for sensitive reports. Do not post private browsing data, access tokens, unpublished manuscripts, or personal information in a public issue.

No product analytics are used on this site.
